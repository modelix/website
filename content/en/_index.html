{{< /blocks/cover >}} {{% blocks/lead color="primary" %}} ## The modelix Platform{#info} ### Our Vision The modelix project develops a next generation platform for domain-specific models on the web. ### Our Mission Modelix applies established web technologies to provide the infrastructure to build domain-specific modeling tools on the web/cloud. The developed infrastructure is a modular platform which integrates with existing language workbenches whilst exploring new approaches. We aim for a productive integration between modern web technologies and best-practices from language engineering. ### Our Origin Our work is inspired by [this document](http://voelter.de/data/pub/APlatformForSystemsAndBusinessModeling.pdf). {{% /blocks/lead %}} {{< blocks/section color="dark" type="features">}} {{% blocks/feature icon="fa-glasses" title="See Modelix in action!" url="http://server.modelix.org/" %}} Check the Demo server runing modelix. {{% /blocks/feature %}} {{% blocks/feature icon="fa-lightbulb" title="Look at Modelix samples!" url="https://docs.modelix.org/modelix/latest/samples/reference/architecture.html" %}} Get started with our example projects and try modelix yourself. {{% /blocks/feature %}} {{% blocks/feature icon="fab fa-slack" title="Join the discussion on Slack!" url="https://jetbrains-mps.slack.com/archives/C01ADCD6VSM" %}} [Sign up](http://slack-mps.jetbrains.com/) to the MPS community Slack and join us in #modelix - or [contact us via E-Mail](mailto:modelix@itemis.com). {{% /blocks/feature %}} {{< /blocks/section >}} {{% blocks/lead color="primary" %}} ### Our State We aim for a productive integration between modern web technologies and best-practices from language engineering. On the path to this final vision, the short-term goal is to use [JetBrains MPS](https://www.jetbrains.com/mps/) as the backend wherever feasible. However, you are not required to use MPS with the modelix platform - we also support other meta-model sources. To this end, we developed several components which allow you to realize a wide range of use-cases with different architectures. Descriptions and information on all individual components can be found [in our documentation](https://docs.modelix.org/). To understand our current state, visit the [architecture section](https://docs.modelix.org/modelix/latest/explanation/architecture.html), to learn more about our intentions, check the [goals section](https://docs.modelix.org/modelix/latest/explanation/goals.html). ### Our Plan If you are interested where we are going, check our [Development, Release, and Roadmap plan](https://docs.modelix.org/modelix/latest/main/roadmap.html). {{% /blocks/lead %}} {{< blocks/section color="dark" type="features">}} {{% blocks/feature icon="fab fa-github" title="Browse Platform Releases!" url="https://github.com/modelix/modelix.platform/tags/" %}} We [release the modelix platform bi-yearly](https://docs.modelix.org/modelix/latest/main/roadmap.html#_release_cycle).
